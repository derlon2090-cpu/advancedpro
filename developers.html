الأنظمة الأساسية

الأنظمة الرئيسية

هذا هو التقسيم الأنسب لبناء المنصة بطريقة قابلة للصيانة والتوسع.

A) نظام المستخدمين

تسجيل، دخول، حالة الحساب، وربط الاشتراك أو الكود بالمستخدم.

  • Auth عبر البريد أو مزود خارجي
  • Roles: user / admin
  • عرض الأرصدة والحالة ونتائج التوليد

B) نظام الأكواد

الأكواد ترتبط بخطة داخلية تحدد حدود الاستخدام ومدة الصلاحية والتجديد.

  • Plan binding
  • Single / Multi redemption
  • Starts on redeem / fixed expiry

C) نظام الباقات والطلبات

عرض باقات عامة للبيع وربط كل طلب مدفوع باشتراك فعلي للمستخدم.

  • Packages catalog
  • Orders + payment tracking
  • Webhook auto-activation لاحقًا

D) نظام التوليد

إرسال الطلبات إلى المزود الخارجي، حفظ الحالة، خصم الرصيد، وحفظ النتائج.

  • Image / Video generations
  • Pending / Processing / Completed / Failed
  • Usage logs + provider logs

E) لوحة الأدمن

إدارة الخطط، الأكواد، الباقات، السجلات، التقارير، واستهلاك المزود الخارجي.

  • Plans, codes, packages
  • Revenue + usage visibility
  • Manual adjustments when needed

F) منطق التجديد الدوري

Job دوري يقرأ الاشتراكات النشطة ويطبق إما topup أو reset حسب نوع الخطة.

  • renewal_mode = topup
  • renewal_mode = reset
  • تحديث next_renewal_at وإيقاف المنتهي
آلية التنفيذ

آلية العمل

المسارات الأساسية التي يجب أن يغطيها الـ Backend من أول التفعيل حتى التوليد.

1) عند إدخال الكود

  • التحقق من وجود الكود وتفعيله وصلاحيته وعدد مرات الاستخدام.
  • إنشاء user_entitlement وربطه بالمستخدم والخطة.
  • نسخ حدود الرصيد ومدة الفيديو وتحديد start_at و end_at.

2) عند شراء باقة

  • إنشاء order بعد الدفع الناجح.
  • ربط الطلب بالباقة وإنشاء entitlement جديد للمستخدم.
  • إضافة الأرصدة مباشرة أو إرسال كود حسب طريقة التشغيل.

3) عند طلب توليد صورة أو مقطع

  • التحقق من الاشتراك الفعال وتوفر الرصيد والمدة المسموحة.
  • إنشاء سجل generation بحالة pending ثم إرسال الطلب للمزود.
  • عند النجاح: حفظ النتائج، خصم الرصيد، وتسجيل الاستخدام.

4) عند وقت التجديد

  • قراءة next_renewal_at لكل اشتراك نشط.
  • تطبيق reset أو topup بناءً على الخطة.
  • زيادة renewals_used وتحديث الدورة القادمة أو إنهاء الاشتراك.
